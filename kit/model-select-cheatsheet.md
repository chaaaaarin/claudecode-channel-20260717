# 🎁 AIモデル選定チートシート（コーディング用）

タスクの性質と予算から、今どのモデルを選ぶべきかを3ステップで判断できるシートです。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: メモアプリ・Notion・テキストファイルなど、いつでも見返せる場所に保存してください。

```text
【AIモデル選定チートシート（コーディング用）】

■ ステップ1：最優先事項を決める
□ 最新・最高品質を求めるタスクか？
→ YES: Claude Fable 5を選ぶ（総合力トップ、コストは最も高い）
□ 予算重視・大量処理をこなしたいか？
→ YES: DeepSeek V4かGLM-5.2を選ぶ（出力コストはKimi K3の数分の一）
□ オープンウェイトで自前ホスト・改造したいか？
→ YES: Kimi K3を選ぶ（2026年7月27日にフルウェイト公開予定）

■ ステップ2：用途で絞る
□ フロントエンド・UI生成が中心か？
→ YES: Kimi K3かFable 5を選ぶ（Frontend Code Arenaの上位2モデル）
□ 長時間・自律的なエージェントタスクが中心か？
→ YES: Kimi K3を優先候補にする（長時間タスクでの評判が高い。ただし低速・高トークン消費に注意）
□ 3D・複雑なインタラクション実装が必要か？
→ YES: Fable 5を優先候補にする（3D系ベンチマークで依然強い）

■ ステップ3：コストで最終確認
□ 出力コストを月内で厳密に管理したいか？
→ YES: 月間想定出力トークン数×各モデルの単価で試算してから決める
指示例: 「想定の月間出力トークン数は◯◯万トークン。この条件でKimi K3・Fable 5・DeepSeek V4のコストを比較して」

■ 価格早見（100万トークンあたり・出力）
Kimi K3: $15／Claude Fable 5: $50／GLM-5.2: $4.40／DeepSeek V4: $0.87

□ 判断に迷ったら？
→ YES: まず安いモデルで試作し、品質が足りない箇所だけ上位モデルに差し替える「バーベル戦略」を検討する
```

**ポイント**: 価格は変動するため、実際に使う直前に各社の公式Pricingページで最新値を確認してください。
